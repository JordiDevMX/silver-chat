# Architecture & Routing

## SPA Architecture

SilverChat is built with **TanStack Start in SPA mode**. Despite TanStack Start's SSR capabilities, the project is configured for static SPA output — no server runtime is deployed. The Vite config (`vite.config.ts`) enables SPA mode with prerendering:

```ts
tanstackStart({
  spa: {
    enabled: true,
    prerender: { enabled: true, crawlLinks: true, retryCount: 3 },
  },
})
```

The build produces static assets in `dist/client/`, which Netlify serves directly. The SSR server bundle is still generated for local dev but unused in production.

### Build & Deploy

| File | Role |
|---|---|
| `vite.config.ts` | Vite + TanStack Start (SPA) + Tailwind + PWA plugin config |
| `netlify.toml` | Build command (`pnpm run build`), publish dir (`dist/client`), SPA fallback redirect to `_shell.html` |

The Netlify SPA redirect ensures all unknown paths serve the prerendered shell:

```toml
[[redirects]]
  from = "/*"
  to = "/_shell.html"
  status = 200
```

### SSR Error Handling (dev only)

`src/server.ts` and `src/start.ts` set up error middleware for SSR during local development. `src/lib/error-page.ts` contains a static HTML error page. `src/lib/error-capture.ts` captures errors that h3 swallows. These files are not used in production (SPA mode) but remain for dev-time robustness.

## Routing (TanStack Router)

Routes are file-based. The route tree is generated into `src/routeTree.gen.ts` — **do not edit this file**; run `pnpm run generate-routes` after adding or modifying routes.

### Router setup (`src/router.tsx`)

```ts
const router = createRouter({
  routeTree,
  context: { queryClient },
  scrollRestoration: true,
  defaultPreloadStaleTime: 0,
});
```

A `QueryClient` instance is passed via router context, making it available to route loaders through `Route.useRouteContext()`.

### Route map

| Path | File | Description |
|---|---|---|
| `/` | `src/routes/index.tsx` | Home — tabbed view (chats, updates, communities, calls) |
| `/chat/$id` | `src/routes/chat.$id.tsx` | Conversation thread for a specific chat |
| `/communities/$communityId/channels/$channelId` | `src/routes/communities.$communityId.channels.$channelId.tsx` | Community channel thread |
| `/about` | `src/routes/about.tsx` | About page |
| (root) | `src/routes/__root.tsx` | Root shell with providers |

### Route loaders & data prefetching

Routes use TanStack Router's `loader` to prefetch data into the React Query cache before rendering:

```ts
// src/routes/chat.$id.tsx
loader: ({ params, context }) => {
  const chat = mockChats.find((c) => c.id === params.id);
  if (!chat) throw notFound();
  context.queryClient.setQueryData(messagesQueryKey(params.id), getMessages(params.id));
  return { chat };
}
```

This pattern is used identically in the community channel route. The loader returns data via `Route.useLoaderData()` and throws `notFound()` from TanStack Router for missing entries.

## Root Shell (`src/routes/__root.tsx`)

The root route sets up the application shell:

1. **No-flash theme script** — an inline `<script>` in `<head>` that reads `localStorage` for theme mode and accent, applies `data-theme` and `theme-*` class before first paint to prevent theme flash.
2. **`QueryClientProvider`** — wraps the app with a shared `QueryClient`.
3. **`ThemeProvider`** — wraps the app with the theme context (`src/hooks/useTheme.tsx`).
4. **PWA store side-effect import** — `import "@/hooks/pwaInstallStore"` registers `beforeinstallprompt` listeners at module init, before React mounts, to avoid losing the event.
5. **`<Outlet />`** — renders the matched child route.
6. **`<Toaster />`** — global toast portal (sonner).

The root also defines `head()` with meta tags (title, viewport, PWA-related meta), favicon links, and the manifest link. It provides custom `notFoundComponent` and `errorComponent` implementations.

## Path Aliases

Two aliases point to `./src/*`:
- `@/*` — used in most imports
- `#/*` — used in a few component imports (e.g., `ChatListItem`)

Both are configured in `tsconfig.json` `paths` and `package.json` `imports`.

## ESLint & TypeScript

- **TypeScript**: strict mode, `noUnusedLocals`, `noUnusedParameters`, `noFallthroughCasesInSwitch`, `verbatimModuleSyntax`, `noEmit` (bundler mode).
- **ESLint**: extends `@tanstack/eslint-config`, disables `import/no-cycle`, `import/order`, `sort-imports`, `@typescript-eslint/array-type`, `@typescript-eslint/require-await`.

## When Changing Architecture

- **Adding routes**: Create a new file in `src/routes/`, then run `pnpm run generate-routes` to regenerate `src/routeTree.gen.ts`. The generated file must not be manually edited.
- **Changing the root shell**: All global providers (QueryClient, Theme, PWA) are in `__root.tsx`. Removing `<Outlet />` breaks all child routes.
- **Build config changes**: The SPA mode config in `vite.config.ts` determines deployment output. If switching away from SPA mode, the Netlify redirect and publish dir must change too.
